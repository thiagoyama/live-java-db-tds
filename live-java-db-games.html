🏭 Padrão Factory — ConnectionFactory

🏭
O que é Factory?

É um padrão de projeto que centraliza a criação de objetos. Em vez de cada classe criar sua própria conexão, todas pedem à fábrica — uma fonte única e confiável.

✅
Por que usar?

Se a URL, usuário ou senha mudar, você corrige em um único lugar. Sem precisar caçar 10 classes diferentes com código duplicado.

🎮 Pense como o servidor central do MMO — todos os jogadores (classes) se conectam pelo mesmo ponto.

O que é o Model?

É a representação em Java da tabela do banco. Cada coluna da tabela vira um atributo da classe — é o objeto que transporta os dados entre todas as camadas.

O que é DAO?

Data Access Object — isola toda a lógica de acesso ao banco em uma classe dedicada. A View não sabe como os dados são salvos; ela só pede ao DAO.

🔌 As Interfaces do JDBC

🔑
DriverManager
Ponto de entrada. Recebe a URL de conexão e as credenciais, retorna uma Connection.
getConnection(url, user, pass)
🔗
Connection
Representa a sessão aberta com o banco. A partir dela criamos os statements e gerenciamos transações.
prepareStatement(sql)
🛡️
PreparedStatement
Executa SQL parametrizado com ?. Previne SQL Injection e é mais eficiente que o Statement simples.
setX() / executeQuery()
📋
ResultSet
Cursor que percorre as linhas retornadas pelo SELECT. Acessa colunas pelo nome.
next() / getX("coluna")
